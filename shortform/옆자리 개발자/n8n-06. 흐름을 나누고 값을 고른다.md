**주문 데이터 30건이 들어왔습니다. 그런데 모든 주문을 같은 방식으로 처리할 수는 없습니다.**
**"processing" 주문만 골라서 테이블에 넣고, 나머지는 다른 처리를 해야 합니다.**
**이번 글에서는 조건에 따라 흐름을 나누는 If 노드와, 필요한 값만 골라내는 Edit Fields 노드를 다룹니다.**

이번 편에서는 하나의 흐름이 어떻게 여러 갈래로 갈라지는지 봅니다.
여기서 중심이 되는 핵심 단어는 `선택`과 `변환`입니다.
현재 값이라는 `상태`를 보고 길을 나누고, 필요한 값만 골라 다음 단계로 넘기는 과정을 확인하겠습니다.

---

### 조건에 따라 흐름을 나눈다

지금까지의 워크플로우는 하나의 길이었습니다.
HTTP Request로 데이터를 가져오고, Airtable에 저장했습니다.

그런데 Nathan은 모든 주문이 아니라 `processing` 상태인 주문만 테이블에 넣고 싶습니다.
`orderStatus`가 `processing`이면 Airtable에 저장하고, 그렇지 않으면 다른 처리를 해야 합니다.

이런 if-then-else 구조를 조건 분기라고 합니다.
n8n에서는 If 노드가 이 역할을 합니다.

> [!NOTE]
> If 노드는 **선택**입니다.

조건에 따라 True와 False, 두 갈래로 길이 갈라집니다.

---

### If 노드를 추가한다

HTTP Request 노드와 Airtable 노드 사이에 If 노드를 넣습니다.

두 노드를 연결하는 화살표 위에 마우스를 올립니다.
**+** 버튼이 나타납니다. 이것을 선택하면 기존 연결이 끊기고, 새 노드를 끼워 넣을 수 있습니다.

If를 검색하고 선택합니다.

---

### If 노드를 설정한다

If 노드는 두 값을 비교합니다.
하나는 데이터에서 가져온 값이고, 다른 하나는 기준값입니다.

**value1 설정**

value1 필드에 `{{ $json.orderStatus }}`를 입력합니다.

이것은 표현식(Expression)입니다.
표현식은 이전 노드의 출력에서 값을 꺼내는 방법입니다.
`$json`은 이전 노드가 출력한 데이터를 가리킵니다.
`$json.orderStatus`는 그 데이터에서 `orderStatus`라는 이름표(key)의 값을 꺼냅니다.

> [!NOTE]
> **입력 → 처리 → 출력** 구조가 여기서도 작동합니다.

이전 노드의 출력이 이 노드의 입력이 됩니다.
`{{ $json.orderStatus }}`는 그 출력에서 값을 꺼내는 것입니다.

표현식을 직접 타이핑하지 않아도 됩니다.
value1 필드 오른쪽의 **Expression** 탭을 선택하고, 링크 아이콘을 눌러 Expression Editor를 엽니다.

<img src="https://raw.githubusercontent.com/bhcha/new_blog/main/shortform/images/l1-c5-5-3-if-node-open-editor.png" alt="Expression Editor 열기">

왼쪽 패널에서 **HTTP Request** > **orderStatus**를 드래그하여 Expression 필드에 놓으면 됩니다.

<img src="https://raw.githubusercontent.com/bhcha/new_blog/main/shortform/images/l1-c5-5-3-if-node-expression-editor.png" alt="Expression Editor">

**비교 연산 설정**

- **Operation**: **String** > **is equal to**를 선택합니다.
- **value2**: `processing`을 입력합니다.

데이터 타입(boolean, date & time, number, string)을 올바르게 선택해야 합니다.
`orderStatus`는 텍스트이므로 String입니다.

---

### 실행 결과를 확인한다

**Execute step**을 선택하여 If 노드를 테스트합니다.

결과가 두 갈래로 나뉩니다.

- **True Branch**: `orderStatus`가 `processing`인 주문들
- **False Branch**: `orderStatus`가 `booked`인 주문들

<img src="https://raw.githubusercontent.com/bhcha/new_blog/main/shortform/images/l1-c5-5-3-if-node-output.png" alt="If 노드 출력 결과">

> [!NOTE]
> 여기서 `orderStatus`의 값 자체는 **상태**입니다.

`processing`인지 `booked`인지에 따라 다음에 할 행동이 달라집니다.
상태가 명확해야 선택이 가능합니다.

---

### True 경로를 Airtable에 연결한다

Nathan이 필요한 것은 `processing` 주문만 테이블에 넣는 것입니다.
따라서 If 노드의 **true** 커넥터를 Airtable 노드에 연결합니다.

연결 후 Airtable 노드를 다시 테스트합니다.
테스트 전에 Airtable에서 기존 행을 모두 삭제해야 깨끗한 결과를 확인할 수 있습니다.

실행하면 30건이 아닌 14건만 저장됩니다.
`processing` 상태인 주문만 걸러졌기 때문입니다.

---

### 모든 필드가 필요한 것은 아니다

Nathan은 이렇게 말합니다.

> 필요한 건 `employeeName`하고 `orderID`뿐인데, 나머지 필드도 그냥 넣어둘까요?

그러지 않는 것이 좋습니다.

불필요한 데이터를 함께 넣으면 처리에 더 많은 자원이 듭니다.
데이터 전송이 느려지고, 저장 공간도 더 차지합니다.
14건에 5개 필드라면 차이가 크지 않을 수 있습니다.
하지만 수천 건에 수십 개 필드가 되면 컬럼 하나가 성능에 영향을 줍니다.

> [!NOTE]
> **입력 → 처리 → 출력**의 관점에서 보면, 처리에 필요한 입력만 주는 것이 맞습니다.

다음 단계에 쓰이지 않는 값까지 넘길 이유가 없습니다.

---

### Edit Fields (Set) 노드로 값을 고른다

필요한 필드만 골라서 다음 노드로 보내는 것이 Edit Fields (Set) 노드의 역할입니다.

> [!NOTE]
> Edit Fields 노드는 **변환**입니다.

데이터의 형태를 바꿉니다. 필요한 항목만 골라내는 것도 변환입니다.

If 노드의 **true** 커넥터와 Airtable 노드 사이에 Edit Fields 노드를 추가합니다.
기존에 했던 것과 같은 방식으로, **+** 버튼을 눌러 끼워 넣습니다.

### Edit Fields 노드를 설정한다

Edit Fields 노드 설정 화면에서 다음을 구성합니다.

- **Mode**: **Manual Mapping**으로 설정합니다.
- **Fields to Set**에 필요한 필드를 추가합니다.
  - 왼쪽 Input 패널에서 **If** > **orderID**를 드래그하여 첫 번째 필드로 넣습니다.
  - **If** > **employeeName**을 드래그하여 두 번째 필드로 넣습니다.
- **Include Other Input Fields**: **false**로 설정합니다.

이 설정이 핵심입니다.
Include Other Input Fields를 false로 두면, 지정한 필드 외에는 모두 제거됩니다.
5개 필드에서 2개만 골라서 다음으로 보냅니다.

**Execute step**을 선택하면 `orderID`와 `employeeName`만 남은 결과를 확인할 수 있습니다.

<img src="https://raw.githubusercontent.com/bhcha/new_blog/main/shortform/images/l1-c5-4-set-node.png" alt="Edit Fields (Set) 노드 결과">

### Airtable에 저장한다

Edit Fields 노드의 출력을 Airtable에 넣기 위해 테이블을 새로 만듭니다.

Airtable에서 새 테이블 `processingOrders`를 생성합니다.
컬럼은 두 개입니다.

- `orderID` (primary field): Number
- `employeeName`: Single line text

기존에 자동 생성된 빈 행은 삭제합니다.

n8n으로 돌아와서 Edit Fields 노드의 커넥터를 Airtable 노드에 연결합니다.
Airtable 노드의 테이블 설정을 `orders`에서 `processingOrders`로 변경합니다.
Airtable 노드를 테스트하여 새 테이블에 데이터가 정상적으로 들어가는지 확인합니다.

### 참고

**If 노드와 Switch 노드**

If 노드는 True/False, 두 갈래만 지원합니다.
만약 조건이 두 가지가 아니라 여러 가지라면 Switch 노드를 사용합니다.
Switch 노드는 여러 개의 출력 커넥터를 가질 수 있습니다.

> [!NOTE]
> **선택**의 확장입니다. 두 갈래가 아니라 여러 갈래로 나뉘는 것입니다.

**표현식(Expression)**

표현식은 프로그래밍 언어에서 값을 만들어내는 문자와 기호의 조합입니다.
n8n에서 표현식은 다른 노드의 데이터를 참조할 때 사용합니다.
`{{ $json.orderStatus }}`처럼 이중 중괄호 안에 작성합니다.
이 예시에서는 If 노드가 HTTP Request 노드의 출력 데이터를 참조합니다.

**Edit Fields 노드의 다른 용도**

Edit Fields 노드는 기존 필드를 골라내는 것 외에도, 완전히 새로운 데이터를 만들 수 있습니다.
기존 값을 덮어쓰는 것도 가능합니다.
이전 노드에서 전달된 데이터를 기대하는 노드(스프레드시트나 데이터베이스에 값을 넣는 경우 등) 앞에서 특히 유용합니다.
