# 첫 번째 워크플로우

**직접 만들어봐야 감이 옵니다. 이번 글에서는 n8n 공식 문서의 두 가지 튜토리얼을 따라갑니다. 2개짜리 미니 워크플로우 하나, 5개짜리 실전 워크플로우 하나. 핵심 단어들이 화면 위에서 어떻게 보이는지 확인합니다. 추가로 공식문서에는 없는 워크플로우도 추가로 만들어 볼 예정입니다.**

이번 편에서는 핵심 단어가 실제 워크플로우 위에서 움직이는 모습을 봅니다.
`입력 → 처리 → 출력`은 더 이상 설명이 아니라 실행 결과로 보이기 시작합니다.
여기에 `Trigger`, `선택`, `명령`, 그리고 아이템이 흐르는 방식까지 함께 확인하겠습니다.

---

### 미니 워크플로우: Hacker News에서 기사 가져오기

첫 번째 워크플로우는 간단합니다.
Hacker News에서 "automation" 키워드가 포함된 기사 10개를 가져옵니다.

노드 2개로 끝납니다.

---

### 1단계. Manual Trigger 노드 추가

캔버스에서 노드 패널을 엽니다. 상단 우측의 **+** 아이콘을 누르거나 키보드에서 **N**을 누릅니다.

**Manual Trigger**를 검색하고 선택합니다.

이 노드는 **Execute Workflow** 버튼을 눌렀을 때 워크플로우를 시작합니다.
수동으로 실행하고 싶을 때 쓰는 트리거입니다.

> Manual Trigger를 건너뛰고 다른 노드를 먼저 추가해도 됩니다. 트리거 없이 노드를 추가하면 n8n이 Manual Trigger를 자동으로 만들어줍니다.

실제 업무에서는 Schedule Trigger 같은 자동 트리거를 쓰는 경우가 더 많습니다.
자동 트리거는 이 글의 두 번째 튜토리얼에서 다룹니다.

---

### 2단계. Hacker News 노드 추가

Manual Trigger 노드 오른쪽의 **+** 아이콘을 누릅니다.

**Hacker News**를 검색하고 선택합니다.
**Actions** 섹션에서 **Get many items**를 선택합니다.

캔버스에 노드가 추가되고, 설정 창이 열립니다.

---

### 3단계. Hacker News 노드 설정

노드 설정 창에는 세 가지 탭이 있습니다.

- **Parameters**: 이 노드가 무엇을 할지 정하는 설정입니다.
- **Settings**: 노드의 표시 방식이나 실행 방식을 정하는 설정입니다.
- **Docs**: 이 노드의 공식 문서를 새 창에서 엽니다.

> Parameters는 노드마다 다릅니다. 노드의 기능에 따라 달라집니다. Settings는 모든 노드에 공통입니다.

**Parameters 설정**

- **Resource**: `All` — 전체 기사를 대상으로 합니다.
- **Operation**: `Get Many` — 여러 건을 가져옵니다.
- **Limit**: `10` — 최대 10건까지만 가져옵니다.
- **Additional Fields > Add Field > Keyword**: `automation` — "automation" 키워드가 포함된 기사만 가져옵니다.

<img src="https://raw.githubusercontent.com/bhcha/new_blog/main/shortform/images/ch2-hacker-news-node-parameters.png" alt="Hacker News 노드 파라미터">

> [!NOTE]
> **입력 → 처리 → 출력** 구조가 여기서 보입니다. 이 Parameters가 바로 Hacker News 노드에게 주는 **입력**입니다. "어떤 데이터를, 몇 개, 어떤 조건으로 가져올 것인가"를 정해주는 것입니다.

**Settings 설정**

- **Notes**: `Get the 10 latest articles.`
- **Display note in flow?**: `true`로 토글

Notes를 작성하면 캔버스에서 노드 아래에 설명이 표시됩니다.
복잡한 워크플로우나 다른 사람과 공유하는 워크플로우에서 특히 유용합니다.

<img src="https://raw.githubusercontent.com/bhcha/new_blog/main/shortform/images/ch2-hacker-news-node-settings.png" alt="Hacker News 노드 설정">

> 노드의 이름을 바꿀 수도 있습니다. 노드를 선택한 상태에서 **F2**를 누르거나, 노드 창에서 이름을 클릭하면 됩니다. 원래 노드 유형은 Settings 하단에서 확인할 수 있습니다.

<img src="https://raw.githubusercontent.com/bhcha/new_blog/main/shortform/images/ch2-renaming-node.png" alt="노드 이름 변경">

### 4단계. 노드 실행

노드 설정 창에서 **Execute step** 버튼을 누릅니다.

결과가 **Output** 영역에 나타납니다. 10건의 기사가 Table 형식으로 보입니다.

<img src="https://raw.githubusercontent.com/bhcha/new_blog/main/shortform/images/ch2-results-table-view.png" alt="실행 결과 Table 뷰">

실행이 성공하면 캔버스의 노드 위에 초록색 체크 표시가 나타납니다.

<img src="https://raw.githubusercontent.com/bhcha/new_blog/main/shortform/images/ch2-successfully-executed.png" alt="실행 성공 화면">

결과는 세 가지 형식으로 확인할 수 있습니다. 노드 창 상단에서 **Table**, **JSON**, **Schema** 중 하나를 선택합니다.

- **Table**: 기본 형식입니다. 행이 각 기사이고, 열이 속성(제목, URL, 점수 등)입니다.
- **JSON**: 같은 데이터를 JSON 형식으로 보여줍니다.
- **Schema**: 데이터의 구조(어떤 속성이 있는지)를 보여줍니다.

<img src="https://raw.githubusercontent.com/bhcha/new_blog/main/shortform/images/ch2-results-json-view.png" alt="실행 결과 JSON 뷰">

이것이 **출력을 확인하는 방법**입니다. 입력(Parameters)을 넣고, 노드가 처리하고, 결과(Output)가 나왔습니다.

> [!NOTE]
> **입력 → 처리 → 출력**이 화면 위에 그대로 보입니다.

Output 영역에는 추가 정보도 있습니다.

- Output 제목 옆의 아이콘에 마우스를 올리면 **Start Time**(실행 시작 시간)과 **Execution Time**(소요 시간)을 확인할 수 있습니다.
- 그 아래에 **10 items**이라고 표시됩니다. Limit를 10으로 설정했기 때문입니다. Limit를 설정하지 않았다면 실제로 몇 건이 반환되었는지 여기서 확인합니다.

> 노드에 빨간색 경고 아이콘이 나타나면 에러가 발생한 것입니다. 크레덴셜이 없거나, Parameters 설정이 잘못되었을 수 있습니다.

<img src="https://raw.githubusercontent.com/bhcha/new_blog/main/shortform/images/ch2-error-node.png" alt="에러 표시">

이 Hacker News 노드는 데이터를 가져오기만 합니다. 원본을 바꾸지 않습니다.

### 5단계. 워크플로우 저장

노드 편집이 끝나면 **Back to canvas**를 눌러 캔버스로 돌아갑니다.

워크플로우 이름을 바꿉니다. 상단의 워크플로우 이름을 클릭하고 "Hacker News workflow"로 변경합니다.

저장하는 방법은 두 가지입니다.

- 키보드에서 `Ctrl + S` (Mac은 `Cmd + S`)
- 우측 상단의 **Save** 버튼 클릭

"Saved"라고 표시되면 자동 저장된 것입니다.

미니 워크플로우가 완성되었습니다.
Manual Trigger가 시작하고, Hacker News 노드가 데이터를 가져왔습니다.
노드 2개짜리 워크플로우지만, **입력 → 처리 → 출력**의 전체 흐름이 들어있습니다.

### 실전 워크플로우: NASA 태양 플레어 리포트

두 번째 워크플로우는 조금 더 복잡합니다.
매주 NASA API에서 태양 플레어 데이터를 가져와서, 등급별로 분류하고, 외부 서비스로 보고서를 보냅니다.

노드 2개입니다. 선택, 질의, 반복이 추가됩니다.

---

### Step 1. 새 워크플로우 만들기

n8n을 열면 다음 중 하나가 보입니다.

- 환영 메시지와 두 개의 버튼이 있는 화면: **Start from Scratch**를 선택합니다.
- Overview 페이지의 워크플로우 목록: **Create Workflow**를 선택합니다.

---

### Step 2. Schedule Trigger 추가

n8n에서 워크플로우를 시작하는 방법은 두 가지입니다.

- **수동**: **Execute Workflow** 버튼을 눌러 직접 실행
- **자동**: Trigger 노드를 첫 번째 노드로 추가하여 자동 실행

이번에는 **Schedule Trigger**를 씁니다. 정해진 시간에 자동으로 실행되는 트리거입니다.

1. **Add first step**을 선택합니다.
2. **Schedule**을 검색하고 **Schedule Trigger**를 선택합니다.
3. **Trigger Interval**: `Weeks`
4. **Weeks Between Triggers**: `1`
5. **Trigger on Weekdays**: `Monday`
6. **Trigger at Hour**: `9am`
7. **Trigger at Minute**: `0`
8. 설정을 마치고 노드를 닫습니다.

매주 월요일 아침 9시에 워크플로우가 자동으로 실행됩니다.

> [!NOTE]
> **반복**입니다. 조건(매주 월요일 9시)이 만족될 때마다 되풀이됩니다.

---

### Step 3. NASA 노드 추가와 크레덴셜 설정

NASA 노드는 NASA의 공개 API를 사용하여 태양 활동 데이터를 가져옵니다.

> **크레덴셜(Credential)**: 외부 서비스에 접근하려면 인증 정보가 필요합니다. n8n에서는 이것을 크레덴셜이라고 부릅니다. API Key, 비밀번호, OAuth 토큰 등이 크레덴셜입니다.

1. Schedule Trigger 노드의 **Add node** 커넥터를 누릅니다.
2. **NASA**를 검색하고 선택합니다.
3. **Get a DONKI solar flare**를 선택합니다. 최근 태양 플레어 보고서를 가져오는 기능입니다.

크레덴셜을 설정합니다.

1. **Credential for NASA API** 드롭다운에서 **Create new credential**을 선택합니다.
2. [NASA APIs](https://api.nasa.gov/)에서 API Key를 발급받습니다. **Generate API Key** 폼을 작성하면 이메일로 Key가 옵니다.
3. 받은 API Key를 n8n의 **API Key** 필드에 붙여넣습니다.
4. **Save**를 누릅니다.

기본적으로 DONKI Solar Flare는 최근 30일 데이터를 가져옵니다.
최근 7일로 제한하기 위해 **Additional Fields**를 설정합니다.

1. **Add field**를 선택합니다.
2. **Start date**를 선택합니다.
3. Start date 옆에서 **Expression** 탭을 선택하고, 확장 버튼을 눌러 표현식 편집기를 엽니다.
4. 다음 표현식을 입력합니다.

```
{{ $today.minus(7, 'days') }}
```

이 표현식은 오늘 날짜에서 7일을 빼서 올바른 형식의 날짜를 만들어줍니다.

<img src="https://raw.githubusercontent.com/bhcha/new_blog/main/shortform/images/tutorial-date-expression.png" alt="날짜 표현식">

값을 가공하는 것입니다.

> [!NOTE]
> **변환**입니다. "오늘"이라는 값을 "7일 전"이라는 값으로 바꿨습니다.

5. 표현식 편집기를 닫습니다.
6. **Execute step**을 눌러 노드를 테스트합니다. NASA API를 호출하고 최근 7일간의 태양 플레어 정보가 **OUTPUT** 영역에 나타납니다.
7. 노드를 닫고 캔버스로 돌아갑니다.

---

### Step 4. If 노드로 분기 만들기

NASA 데이터를 등급별로 분류합니다.
태양 플레어에는 다섯 가지 등급이 있습니다. 가장 높은 등급(X)과 나머지를 분리합니다.

1. NASA 노드의 **Add node** 커넥터를 누릅니다.
2. **If**를 검색하고 선택합니다.

If 노드를 설정합니다.

1. NASA 노드의 출력 데이터에서 `classType`을 **Value 1**로 드래그합니다.

> 이전 단계에서 NASA 노드를 실행해야 합니다. 실행하지 않았다면 데이터가 표시되지 않습니다.

2. 비교 연산을 **String > Contains**로 변경합니다.
3. **Value 2**에 `X`를 입력합니다.
4. **Execute step**으로 테스트합니다. 조건에 맞는 데이터는 **true**, 맞지 않는 데이터는 **false**로 분류됩니다.

> 최근 7일간 X등급 플레어가 없을 수 있습니다. 그 경우 Value 2를 `A`, `B`, `C`, `M` 중 하나로 바꿔보세요.

> [!NOTE]
> 이것이 **선택**입니다. 조건에 따라 데이터가 두 갈래로 나뉩니다. "X등급이면 이쪽, 아니면 저쪽."

노드를 닫고 캔버스로 돌아갑니다.

---

### Step 5. Postbin 노드로 결과 보내기

분류된 데이터를 외부 서비스로 보냅니다.
Postbin은 데이터를 받아서 임시 웹 페이지에 표시해주는 서비스입니다.

**true 경로 (X등급 플레어)**

1. If 노드의 **true** 커넥터에서 **Add node**를 누릅니다.
2. **PostBin**을 검색하고 선택합니다.
3. **Send a request**를 선택합니다.
4. [Postbin](https://www.toptal.com/developers/postbin/)에서 **Create Bin**을 눌러 Bin을 만듭니다. 탭을 닫지 마세요.
5. Bin ID를 복사합니다. `1651063625300-2016451240051` 같은 형식입니다.
6. n8n의 **Bin ID** 필드에 붙여넣습니다.
7. **Bin Content** 옆에서 **Expression** 탭을 선택하고, 확장 버튼을 눌러 표현식 편집기를 엽니다.
8. If 노드의 출력에서 `classType`을 표현식 편집기로 드래그합니다. `{{$json["classType"]}}`이라는 참조가 자동으로 생성됩니다.
9. 메시지를 추가하여 다음과 같이 만듭니다.

```
There was a solar flare of class {{$json["classType"]}}
```

<img src="https://raw.githubusercontent.com/bhcha/new_blog/main/shortform/images/tutorial-expression-output.png" alt="표현식 결과">

10. 표현식 편집기를 닫고, Postbin 노드를 닫습니다.

**false 경로 (나머지 등급)**

1. Postbin 노드에 마우스를 올리고 **Node context menu > Duplicate node**를 선택합니다.
2. If 노드의 **false** 커넥터를 복제된 Postbin 노드의 왼쪽으로 연결합니다.

Postbin 노드는 외부 서비스에 데이터를 보냅니다. 외부의 상태를 바꿉니다.

> [!NOTE]
> 외부의 상태를 바꾸는 것이므로 **명령**입니다.

> Postbin의 Bin은 생성 후 30분이 지나면 만료됩니다. 시간이 초과되면 새 Bin을 만들고 Bin ID를 업데이트하세요.

---

### Step 6. 전체 워크플로우 실행

1. **Execute Workflow** 버튼을 누릅니다. 전체 워크플로우가 순서대로 실행됩니다.
2. Postbin 탭으로 돌아가 새로고침합니다. 결과가 표시됩니다.
3. 이 워크플로우를 매주 자동으로 실행하려면 **Publish** 버튼을 눌러 활성화합니다.

<img src="https://raw.githubusercontent.com/bhcha/new_blog/main/shortform/images/tutorial-first-workflow-complete.png" alt="완성된 워크플로우">

> [!NOTE]
> 이 예시는 워크플로우 자동화의 기본 구조를 한 번에 보여줍니다.
> Trigger로 시작하고, 외부에서 데이터를 가져오고, 조건으로 나누고, 외부 서비스에 결과를 보냅니다.

### 실전 워크플로우: 네이버 뉴스 가지고 오기

공식문서에 없는 워크플로우를 해보겠습니다. 이 워크플로우는 나중에 다른 예시에서 계속 덧붙여 볼 예정입니다. 

매일 아침 네이버 뉴스 API에서 제가 일하는 산업의 데이터를 가져와 보겠습니다. 노드 2개입니다. 굉장히 간단하지만 실용적이고 네이버뉴스 해당 기사가 있기만 하다면 많은 회사에서 본인 회사 뉴스를 크롤링하는것과 크게 차이가 없을 것 입니다.

---

### Step 1. Schedule Trigger 추가

다시한번 **Schedule Trigger**를 씁니다.
정해진 시간에 자동으로 실행되는 트리거입니다.
이번엔 조금 색다른 방법으로 해보겠습니다.

1. **Add first step**을 선택합니다.
2. **Schedule**을 검색하고 **Schedule Trigger**를 선택합니다.
3. **Trigger Interval**: `Custom (Cron)`
4. **Expression**: `0 0 * * 1-5`
5. 설정을 마치고 노드를 닫습니다.

매주 월~금요일 매일 아침 8시에 워크플로우가 자동으로 실행됩니다.

> [!NOTE]
> Trigger에 대한 설명은 앞에서 했기때문에 넘어 가겠습니다.
> 이번 트리거는 특별히 Cron이라는 것을 사용했는데 비개발자분들한테 매우 낯선 내용일겁니다. Cron은 리눅스/유닉스 운영체제에서 정해진 시간에 자동으로 명령어나 스크립트를 실행해주는 스케줄러로 Crontab 문법은 여러곳에서 쓰여 알아두시는게 좋을것 같아 해봤습니다.

### Step 2. 네이버 뉴스 API 호출하기

네이버 뉴스 API를 가지고 오는 법은 따로 다루지 않겠습니다.
API를 사용하려면 `X-Naver-Client-Id`와 `X-Naver-Client-Secret`
두가지가 필요합니다. 미리 준비해주세요.

1. Schedule Trigger 노드의 **Add node** 커넥터를 누릅니다.
2. **HTTP Request**를 검색하고 선택합니다.
3. 상단의 Import cURL을 선택합니다. 아래내용을 복사 붙여넣기 합니다.
<img src="https://raw.githubusercontent.com/bhcha/new_blog/main/shortform/images/Pasted%20image%2020260324232110.png" alt="Pasted image 20260324232110.png">
```sh
curl "https://openapi.naver.com/v1/search/news.xml?display=99&start=1&sort=sim" \
    -H "X-Naver-Client-Id: {애플리케이션 등록 시 발급받은 클라이언트 아이디 값}" \
    -H "X-Naver-Client-Secret: {애플리케이션 등록 시 발급받은 클라이언트 시크릿 값}" -v
```


> [!NOTE]
> cURL은 Client URL의 약자로, 커맨드라인에서 URL로 데이터를 주고받는 도구 입니다. URL뒤에 물음표 부터는 Query Parameter 또는 Query String 으로 혼용해서 사용됩니다. 해당 API에 조건 같은것들을 포함시킬수 있습니다. 예를들어, 특적 산업의 뉴스만 가지고 오고 싶으면 ?query=제약 이렇게 추가하면 됩니다.
> 이것도 많이 쓰입니다. 알아두면 좋습니다.

여기까지가 네이버 뉴스를 가지고 오는 전부입니다. 이 워크플로우는 나중에 고도화 해보겠습니다.

### 세 개의 워크플로우에서 본 것

미니 워크플로우는 노드 2개였습니다.
Manual Trigger가 시작하고, Hacker News가 데이터를 가져왔습니다.
**입력 → 처리 → 출력**이 눈에 보였습니다.

NASA 워크플로우는 노드 5개였습니다.
여기에는 더 많은 것이 들어있습니다.

- **Schedule Trigger**: 정해진 시각에 워크플로우를 시작합니다. **입력**이자 자동 실행을 위한 **Trigger**입니다.
- **NASA 노드**: 데이터를 가져옵니다. **질의**입니다.
- **표현식** `{{ $today.minus(7, 'days') }}`: 값을 가공합니다. **변환**입니다.
- **If 노드**: 등급에 따라 두 갈래로 나눕니다. **선택**입니다.
- **Postbin 노드**: 외부 서비스에 데이터를 보냅니다. **명령**입니다.

네이버 뉴스를 가지고 오는 워크플로우는 노드 2개였습니다.
간단하지만 실용적입니다.

매주 같은 시각에 이 흐름이 다시 시작된다는 점에서는 **반복**도 함께 보입니다.
하지만 경로를 둘로 나누는 선택은 If 노드에서 일어납니다.

카페 비유로 설명한 것들이 노드 위에 그대로 올라가 있습니다.

### 참고

**크레덴셜(Credential)**
외부 서비스에 접근하기 위한 인증 정보입니다. API Key, 비밀번호, OAuth 토큰 등이 해당합니다. n8n에서는 크레덴셜을 한 번 등록하면 여러 워크플로우에서 재사용할 수 있습니다. 크레덴셜은 핵심 단어 4묶음에 직접 매핑되지는 않지만, 외부 서비스와의 연결에 필수적인 개념입니다.

**표현식(Expression)**
n8n에서 값을 동적으로 만들기 위한 문법입니다. `{{ }}` 안에 작성합니다. 이전 노드의 출력값을 참조하거나(`{{$json["classType"]}}`), 날짜를 계산하거나(`{{ $today.minus(7, 'days') }}`), 텍스트를 조합할 수 있습니다. 고정된 값 대신 "상황에 따라 달라지는 값"을 만들 때 씁니다.

**Postbin**
데이터를 받아서 임시 웹 페이지에 보여주는 테스트용 서비스입니다. 실제 업무에서는 이메일, Slack, Google Sheets 같은 서비스를 대신 연결합니다.


이 세 가지는 핵심 단어 4묶음에 직접 매핑되지 않습니다. 핵심 단어가 소프트웨어의 구조를 설명하는 것이라면, 크레덴셜, 표현식, Postbin은 그 구조 위에서 실제로 쓰이는 도구에 가깝습니다.

하나하나 핵심 단어로 분해하면 할 수 있지만, 억지스러워집니다. 그래서 별도로 설명을 뺐습니다.

이런 용어들은 자동화를 하거나 시스템을 다룰 때 반복해서 나옵니다. 지금 외울 필요는 없습니다. 앞으로 계속 만나게 됩니다.

핵심 단어들이 이해되셨다면, 이 설명들도 어렴풋이 읽히실 겁니다. 그 정도면 다음 글부터 훨씬 수월해집니다.
