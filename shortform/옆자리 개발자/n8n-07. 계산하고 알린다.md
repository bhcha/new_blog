# 계산하고 알린다

> **옆자리 개발자** — 워크플로우 자동화와 바이브 코딩

---

**Booked 주문의 합계를 계산하고, 그 결과를 Discord로 팀에 알립니다. Code 노드로 값을 변환하고, Discord 노드로 외부에 명령을 보내는 과정입니다.**

이번 편에서는 여러 아이템을 받아 계산하고, 그 결과를 밖으로 보내는 과정을 봅니다.
핵심은 `변환`과 `명령`입니다.
여기에 아이템을 하나씩 처리할지, 한꺼번에 처리할지라는 `반복`의 관점도 함께 보겠습니다.

---

### 다음 단계는 계산입니다

이전 글에서 If 노드로 주문을 나누었습니다.
`Booked` 주문만 따로 모인 상태입니다.

Nathan이 다음으로 해야 할 일은 두 가지입니다.

- Booked 주문이 총 몇 건인지
- Booked 주문의 금액 합계가 얼마인지

이 계산을 n8n에서는 **Code 노드**로 처리합니다.

---

### Code 노드는 JavaScript를 직접 쓸 수 있는 노드입니다

n8n의 대부분의 노드는 설정만으로 동작합니다.
하지만 직접 계산을 해야 할 때는 코드를 써야 합니다.

Code 노드는 JavaScript(또는 Python)를 작성할 수 있는 노드입니다.
워크플로우 안에서 데이터를 가공하거나, 여러 값을 합치거나, 원하는 형태로 바꿀 때 사용합니다.

> [!NOTE]
> **변환**입니다.

여러 주문의 금액이라는 재료를, 하나의 합계라는 결과물로 바꿉니다.

---

### Code 노드에는 두 가지 모드가 있습니다

Code 노드는 데이터를 처리하는 방식에 따라 두 가지 모드를 제공합니다.

**Run Once for All Items** — 모든 아이템을 한꺼번에 처리합니다.
주문이 5건이면, 5건을 한 번에 받아서 코드를 한 번 실행합니다.

**Run Once for Each Item** — 아이템마다 한 번씩 처리합니다.
주문이 5건이면, 코드가 5번 실행됩니다.

> [!NOTE]
> **반복**과 관련이 있습니다.

"Run Once for All Items"는 배열 전체를 한 번에 받아서 처리합니다. 반복을 n8n이 대신 처리한 것입니다.
"Run Once for Each Item"은 아이템마다 한 번씩 실행됩니다. 이것은 반복 그 자체입니다.

Nathan의 목적은 전체 합계를 구하는 것이니, **Run Once for All Items**를 선택합니다.

---

### n8n의 데이터 구조를 알아야 합니다

Code 노드를 쓰기 전에, n8n 안에서 데이터가 어떤 형태로 오가는지 알아야 합니다.

n8n에서 노드 사이를 흐르는 데이터는 다음과 같은 구조입니다.

```
[
  {
    "json": {
      "apple": "beets",
      "carrot": {
        "dill": 1
      }
    }
  },
  ...
]
```

배열(`[]`) 안에 객체(`{}`)가 들어있고, 각 객체 안에는 반드시 `json`이라는 키가 있습니다.
실제 데이터는 이 `json` 안에 담깁니다.

> [!NOTE]
> 값의 형태를 설명할 때 나왔던 **배열**과 **객체**가 그대로 보입니다.

배열은 순서가 있는 목록이었고, 객체는 이름표가 붙은 값들의 묶음이었습니다.

n8n의 데이터는 `배열 안의 객체`입니다.
주문 목록이라면, 배열 안에 주문 객체가 하나씩 들어있는 구조입니다.

> `json` 외에 `binary`라는 키도 올 수 있습니다. 이미지나 파일 같은 이진 데이터를 담는 용도이지만, 대부분의 워크플로우에서는 쓸 일이 없습니다.

### 합계를 계산하는 코드를 작성합니다

워크플로우에서 If 노드의 `false` 브랜치(Booked 주문)에 Code 노드를 연결합니다.

Code 노드의 설정은 다음과 같습니다.

- **Mode**: Run Once for All Items
- **Language**: JavaScript

코드 영역에 아래 내용을 붙여넣습니다.

```javascript
let items = $input.all();
let totalBooked = items.length;
let bookedSum = 0;

for (let i=0; i < items.length; i++) {
  bookedSum = bookedSum + items[i].json.orderPrice;
}

return [{ json: {totalBooked, bookedSum} }];
```

코드가 하는 일을 한 줄씩 보겠습니다.

`$input.all()`은 이 노드로 들어온 모든 아이템을 가져옵니다.
Booked 주문 5건이 들어왔다면, 5개의 객체가 담긴 배열을 받는 것입니다.

`items.length`는 배열의 길이, 즉 주문 건수입니다.
`for` 구문은 배열을 하나씩 돌면서 `orderPrice`를 더합니다.

마지막 줄이 중요합니다.

```javascript
return [{ json: {totalBooked, bookedSum} }];
```

결과를 `[{ json: {...} }]` 형태로 반환합니다.
n8n의 데이터 구조에 맞춰서 돌려주는 것입니다.

이것이 Code 노드의 **출력**입니다.

> [!NOTE]
> 입력 → 처리 → 출력의 구조가 여기서도 동일합니다.

여러 건의 주문이 입력으로 들어오고, 코드가 합산을 처리하고, 합계가 담긴 하나의 객체가 출력으로 나갑니다.

> 이 형태를 지키지 않으면 `Error: Always an Array of items has to be returned!` 에러가 발생합니다. n8n은 항상 배열을 기대합니다.

Execute step을 누르면, `totalBooked`(주문 건수)와 `bookedSum`(금액 합계)이 담긴 결과를 확인할 수 있습니다.

### 이제 팀에 알립니다

합계가 나왔으니, Nathan의 다음 단계는 이 결과를 팀 Discord에 공유하는 것입니다.

Code 노드 뒤에 **Discord 노드**를 연결합니다.

Discord 노드를 추가할 때, **Message Actions** 중 **Send a Message**를 선택합니다.

> [!NOTE]
> **명령**입니다.

Discord라는 외부 서비스의 채팅방에 메시지를 보내는 것이니, 외부의 상태를 바꾸는 행위입니다.

---

### Discord 노드를 설정합니다

Discord 노드의 설정은 다음과 같습니다.

- **Connection Type**: Webhook
- **Credential for Discord Webhook**: 새 크레덴셜을 만들고, Webhook URL을 입력합니다
- **Operation**: Send a Message

> Webhook URL은 Discord 채널 설정에서 만들 수 있습니다. n8n 공식 과정에서는 수강 등록 시 받은 URL을 사용합니다.

**Message** 필드에서 Expression 탭을 선택하고, 아래 내용을 입력합니다.

```
This week we've {{$json["totalBooked"]}} booked orders with a total value of {{$json["bookedSum"]}}.
```

`{{$json["totalBooked"]}}`는 Code 노드의 출력에서 `totalBooked` 값을 가져오는 표현식입니다.
이전 노드의 출력이 다음 노드의 입력이 되는 구조입니다.

> [!NOTE]
> 입력 → 처리 → 출력의 체인이 여기서도 이어집니다.

Code 노드의 출력이 Discord 노드의 입력이 됩니다.
Discord 노드는 이 값을 메시지에 넣어서 채널에 전송합니다.

Execute step을 누르면, Discord 채널에 메시지가 도착합니다.

> Discord 대신 Slack이나 Mattermost 같은 다른 커뮤니케이션 앱 노드를 사용할 수도 있습니다. 어떤 노드를 쓰든, 외부에 메시지를 보낸다는 점에서 명령인 것은 같습니다.

### 정리하며

Booked 주문의 합계를 계산하고, 그 결과를 Discord로 팀에 알렸습니다.

Code 노드는 **변환**이었습니다.
여러 건의 주문 금액을 하나의 합계로 바꿨습니다.
결과는 `[{ json: {totalBooked, bookedSum} }]` 형태로, n8n의 데이터 구조(배열 안의 객체)를 그대로 따릅니다.

Discord 노드는 **명령**이었습니다.
외부 서비스에 메시지를 보내서, 채팅방이라는 외부 상태를 바꿨습니다.

Code 노드의 출력이 Discord 노드의 입력이 되는 것은, 입력 → 처리 → 출력의 체인입니다.

> [!NOTE]
> 구조가 노드를 연결할 때마다 반복되고 있습니다.
